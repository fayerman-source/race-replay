## What is currently in progress locally

### 1. World Athletics geometry refactor

Goal:
- make early-race placement physically coherent so lane number does not create fake forward gain/loss

Current local direction:
- [js/utils.js](/home/fayerman/race-replay/js/utils.js) now has a more explicit segment-based track model for:
  - top curve
  - left straight
  - bottom curve
  - right straight
- the model distinguishes:
  - common race-station projection
  - lane-path-distance projection
- [js/race-model.js](/home/fayerman/race-replay/js/race-model.js) computes pre-break placement from lane-path geometry and then blends toward common race-station placement through the merge
- [js/app.js](/home/fayerman/race-replay/js/app.js) renders directly from `runnerState.trackPosition` instead of adding its own path shortcut math

### 2. Public demo picker

## The main unresolved issue

The World Athletics replay still needs manual validation around the opening phase.

The exact unresolved question:
- does the current station-based pre-break model now look physically truthful on the first home straight and through the merge, or is there still a visible mismatch?

Most recent user-reported symptoms before handoff:
- lane 1 was appearing in front incorrectly
- after one fix, Keely seemed to catch up unrealistically from the outside
- the user wants the opening formation to be governed by physics/geometry, not by preference or a visual hack

Most recent local attempt to address that:
- replaced the old forward-offset model with:
  - lane-path-distance pre-break placement
  - common station post-break placement
  - direct rendering from model positions

## Useful facts captured during debugging

- World Athletics replay currently uses:
  - `lane_count: 6`
  - `break_distance_m: 160`
  - `merge_complete_distance_m: 180`
- user’s video review suggested the field started merging inward at about `22.1s`
- local numeric checks showed the geometry is materially healthier than the older model, but **browser-level visual validation still matters**
